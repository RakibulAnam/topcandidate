# Dart / Flutter standards

The mobile app's authoritative coding rules live at:
- [`apps/mobile/AGENTS.md`](../../apps/mobile/AGENTS.md) — spec-driven development model, where to add code, isolate boundary rules.
- [`apps/mobile/analysis_options.yaml`](../../apps/mobile/analysis_options.yaml) — linter config (inherits from `flutter_lints`).
- [`apps/mobile/spec/`](../../apps/mobile/spec/) — the spec documents are normative; the implementation conforms to them.

This file exists to point you at those.
